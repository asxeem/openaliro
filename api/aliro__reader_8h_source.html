5 * aliro_reader — Aliro reader session/transaction layer. Owns the per-connection
6 * Aliro transaction on top of the aliro_ble transport: session lifecycle, the
7 * credential-auth exchange (AUTH0/AUTH1/EXCHANGE), the reader identity and
8 * credential trust gate, the M1-M4 ranging setup, and the handoff of the derived
9 * URSK plus negotiated ranging parameters to the UWB engine.
10 *
11 * Crypto lives in the aliro_crypto component; the wire codec in aliro_apdu.c and
12 * the ranging setup in aliro_ranging.c. See README.md for the flow and
13 * components/aliro_ble/SPEC.md for the transport contract.
14 */

49/* Send the phone a "Reader Status Changed" SDU (Aliro transaction step 23) over the
50 * active ranging session's BleSK channel: `unsecured` true on an approach grant (this
51 * is what fires the iPhone Wallet unlock animation), false on relock. Safe to call
52 * from any task -- it marshals the send onto the BLE-host task. No-op if no ranging
53 * session is established. */

56/* Copy out the credential public key (uncompressed P-256, 65 bytes) of the most
57 * recent session that passed the trust check. The Matter door lock resolves it to
58 * the user that owns it, so the LockOperation event names who unlocked; without
59 * that the event is anonymous and Apple Home notifies every device in the home,
60 * including the one that just unlocked. Returns true if a credential has
61 * authenticated since boot (cred_pub written), false otherwise (left untouched).
62 * Safe to call from any task. */
63bool aliro_reader_authenticated_credential(uint8_t cred_pub[65]);
